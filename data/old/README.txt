This repertory keep a backup of modified files. 

The structure is  :
 
- For redefined meter(s) index(es)
r_YYYYMMDD-stamp.csv, restore into data data/csv/

- For corrected daily value(s) :
c_YYYYMMDD-stamp.csv, restore into data data/meters/

- For decommissioned meter :
d_YYYYMMDD-met_num-stamp.csv, restore into data data/csv/


YYYYMMDD = Date of the original file
stamp = Tag of the modified file 
met_num = Decommissioned meter number
